Démarrer Postgres (port hôte 5433)
PowerShell

cd .\db
docker compose up -d
Vérifier rapidement :

PowerShell

docker run --rm --network db_default -e PGPASSWORD=poc postgres:16 `
  psql -h db-db-1 -U poc -d pocdb -c "select 1;"

→ doit renvoyer 1.

Vérifier backend/src/main/resources/application.properties (déjà bon normalement) :
properties
spring.datasource.url=jdbc:postgresql://localhost:5433/pocdb
spring.datasource.username=poc
spring.datasource.password=poc
spring.jpa.hibernate.ddl-auto=update
spring.jpa.show-sql=true
spring.jpa.database-platform=org.hibernate.dialect.PostgreSQLDialect
Lancer le backend
PowerShell
cd ..\backend
./mvnw clean spring-boot:run
Attendre “Tomcat started on port 8080”.

Tester les endpoints (dans un autre terminal)
PowerShell
Invoke-RestMethod -Uri "http://localhost:8080/api/messages" `
  -Method Post -ContentType 'application/json' -Body '{"text":"hello"}'

Invoke-RestMethod -Uri "http://localhost:8080/api/messages" -Method Get

Voici le minimum pour tout lancer :

cd C:\Users\<toi>\poc-svelte-spring-postgres\db

2) Backend Spring

PowerShell

3) Front Svelte

PowerShell
cd ..\frontend
npm install          # première fois seulement
echo VITE_API_URL=http://localhost:8080/api > .env   # si pas déjà présent
npm run dev -- --host
